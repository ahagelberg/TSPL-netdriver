How templates work

The Elements field is a JSON array. Each item must have a type. Order matters: later elements draw on top of earlier ones (put box before text to frame text).

  • "type": "text" — x, y (top-left, mm from label origin), font (e.g. "3"), content with optional placeholders {{name}} filled from API / test data.
  • "type": "box" — rectangle outline: x, y (top-left), width, height, line_width (border thickness, mm). Emitted as TSPL BOX (thickness converted to dots from printer DPI).

Example (border then text):

[
  { "type": "box", "x": 1, "y": 1, "width": 38, "height": 28, "line_width": 0.25 },
  { "type": "text", "x": 3, "y": 5, "font": "3", "content": "{{line1}}" }
]

Placeholder keys required for printing must appear in Test data or in the API data object. Printers use each printer’s TEXT payload encoding for text. Very limited firmware may not implement BOX; use only text elements if the printer errors on borders.
